There are three types of variables for terraform code


Input variables

Input variables are values I give to Terraform so I don't have to hardcode the 
same information repeatedly. They make the code reusable and easier to change.
Input Variable = Value provided by the user

Instead of this 

resource "aws_instance" "web" {
  ami           = "ami-123456"
  instance_type = "t2.micro"
}

you create an input variable:

variable "instance_type" {
  default = "t2.micro"
}

Then use it 

resource "aws_instance" "web" {
  ami           = "ami-123456"
  instance_type = var.instance_type
}


This helps if you want to create a new instance of same type across your infrastructur
and let say in future if you want to update the instance_type then you simply
chnage the one thing in variable and it will apply to all functions in which it 
uses the variable.




Local Variable

Local variables are helper values that Terraform creates or calculates 
inside the code. They help avoid repeating text and make naming and 
configuration easier to manage.
Local = Helper value Terraform calculates once and reuses

Let's say your company wants all resources named:

project-environment 

for example:
hotel-prod
hotel-dev
hotel-test

variable "project" {
  default = "hotel"
}

variable "environment" {
  default = "prod"
}

locals {
  resource_name = "${var.project}-${var.environment}"
}


Terraform calculates:

hotel-prod


and same with other two and Now you can use it everywhere in resources:

resource "aws_instance" "web" {
  tags = {
    Name = local.resource_name
  }
}


Without locals you would keep typing:

hotel-prod
hotel-prod
hotel-prod

over and over.


This can help you create your customized text input that you desire by either 
using variables or the text and make your own text which you can use everywhere
in the code.




Output Variable 

Output variables are values Terraform returns after creating the 
infrastructure, such as IP addresses, DNS names, resource IDs, or 
endpoints that I may need later.

Output = Information Terraform gives back to you

Suppose Terraform creates an EC2 instance.

AWS gives it a public IP:

54.210.100.50

You want Terraform to show it.

output "public_ip" {
  value = aws_instance.web.public_ip
}

After:

terraform apply

Terraform displays:

Apply complete!

Outputs:

public_ip = 54.210.100.50

Now you know how to connect:

ssh ec2-user@54.210.100.50


By creating the output variable function you can define it on what you want
to know after it creates the infrastructur.
This will help you know specific things you want to know about the infrastructur
once it is running.
